## What is KMP
KMP (Knuth-Morris-Pratt) is an efficient string searching algorithm. Its core idea is to use partial matching information to avoid repeated comparisons and improve search efficiency.

## Three Core Ideas
First, look at the KMP code: (Test link: https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/)
```c++
#include<string>
#include<vector>
using namespace std;
class Solution {
public:
    vector<int> next;
    int strStr(string haystack, string needle) {
        int n = haystack.size();
        int m = needle.size();
        int x = 0, y = 0;   // indices for haystack and needle
        vector<int> next(m);
        // O(m)
        build(next, needle, m);
        // O(n)
        while (x < n && y < m) {
            if (haystack[x] == needle[y]) {
                x++;
                y++;
            } else if (y == 0) {   
                x++;
            } else {
                y = next[y];
            }
        }
        return y == m ? x - y : -1;  // starting index is x - y
    }

    // How to quickly generate the next array
    void build(vector<int>& next, string& needle, int& m) {
        if (m == 1) {
            next[0] = -1;
            return;
        }
        next[0] = -1;
        next[1] = 0;
        int i = 2, cn = 0;
        //  i(m)   i - cn(m)
        //  up        up
        //  unchanged up
        //  up        up
        while (i < m) {
            if (needle[i - 1] == needle[cn]) {  // start from needle index 2-1, initially check if needle[1] == needle[0]
                // subsequent cases where it equals the previous value
                next[i++] = ++cn;                       // length can be at most one more than before, when equal to the value at the previous maximum position
            } else if (cn > 0) {   // cn > 0: current position can still jump forward
                cn = next[cn];
            } else {
                next[i++] = 0; // jumped to the beginning without finding a match
            }
        }
    }
};
```
I will describe its core ideas based on this code.
### Core Idea 1: Assignment of next indices
The generation of the next array is the core part of the KMP algorithm. It records the length of the longest equal prefix and suffix for each position in the pattern string. With this array, the KMP algorithm can quickly skip unnecessary comparisons when a match fails, thereby improving efficiency. Therefore, the values stored at different indices of the next array are meaningful:
* next[i] = x: corresponds one-to-one with needle[i]; x is the length of the longest equal prefix and suffix of the substring before needle[i], and the prefix and suffix cannot be the entire preceding string.
* next[0] = -1: The first character has no prefix or suffix, so a negative value is stored to indicate this.
* next[1] = 0: The prefix and suffix cannot be the entire string, so it is 0.
* The same logic applies for the rest.

### Core Idea 2: How the main KMP logic uses next to accelerate matching
```c++
    while (x < n && y < m) {
        if (haystack[x] == needle[y]) {
            x++;
            y++;
        } else if (y == 0) {   
            x++;
        } else {
            y = next[y];
        }
    }
```
Suppose we have a substring `aabaaf`, and we generate its next array:
* next[0] = -1
* next[1] = 0
* next[2] = 1
* next[3] = 0
* next[4] = 1
* next[5] = 2
We can easily see: \
 When y = 4, the string before needle[4] is `aaba`. needle[next[4] - 1] is the same as needle[3]. At this point, we can change the comparison of needle[4] with haystack[x] to comparing needle[next[4]] (needle[1]) with haystack[x]. This avoids comparing needle[0] again, thus improving speed.


### Core Idea 3: Using cn to accelerate next array generation
```c++
    while (i < m) {
        if (needle[i - 1] == needle[cn]) {  // start from needle index 2-1, initially check if needle[1] == needle[0]
            // subsequent cases where it equals the previous value
            next[i++] = ++cn;                       // length can be at most one more than before, when equal to the value at the previous maximum position
        } else if (cn > 0) {   // cn > 0: current position can still jump forward
            cn = next[cn];
        } else {
            next[i++] = 0; // jumped to the beginning without finding a match
        }
    }
```
Why can cn accelerate the generation of the next array: \
next[0] = -1 and next[1] = 0 are already initialized and not included in the process.
When needle[i - 1] equals needle[cn], the value stored in next[i] is one more than before, because when they are equal, the prefix and suffix can both increase by one compared to before. When they are not equal, if cn > 0, there may exist a prefix that matches the current suffix, so we can replace the suffix with the prefix, achieving accelerated generation of next.


## FAQ
### Why not initialize the array with next[0] = 0, next[1] = 1?
next[0] = 0, next[1] = 1 is the standard version of KMP, while next[0] = -1, next[1] = 0 is the `failure function` version, which can better handle the index relationships of next and allows using cn to accelerate the construction of the next array. It is superior to the standard version.
