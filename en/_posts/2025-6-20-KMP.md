## What is KMP
KMP (Knuth-Morris-Pratt) is an efficient string searching algorithm. Its core idea is to use partial matching information to avoid repeated comparisons and improve search efficiency.

## Three Core Ideas
First, look at the KMP code: (Test link: https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/)
```c++
#include<string>
#include<vector>
using namespace std;
class Solution {
public:
    vector<int> next;
    int strStr(string haystack, string needle) {
        int n = haystack.size();
        int m = needle.size();
        int x = 0, y = 0;   // haystack, needle的下标
        vector<int> next(m);
        // O(m)
        build(next, needle, m);
        // O(n)
        while (x < n && y < m) {
            if (haystack[x] == needle[y]) {
                x++;
                y++;
            } else if (y == 0) {   
                x++;
            } else {
                y = next[y];
            }
        }
        return y == m ? x - y : -1;  // 开头是x - y下标
    }

    // next数组如何快速生成
    void build(vector<int>& next, string& needle, int& m) {
        if (m == 1) {
            next[0] = -1;
            return;
        }
        next[0] = -1;
        next[1] = 0;
        int i = 2, cn = 0;
        //  i(m)   i - cn(m)
        //  上       上
        //  不变     上
        //  上       上
        while (i < m) {
            if (needle[i - 1] == needle[cn]) {  // 从needle中的下标为2 - 1开始, 开始为neeedle[1] == needle[0]相等的情况下
                // 后面为和上一个值相等的情况
                next[i++] = ++cn;                       // 长度最多比之前多一个, 在与之前最大位置的值相等的情况下
            } else if (cn > 0) {   // cn > 0 : 当前位置还能往前蹦
                cn = next[cn];
            } else {
                next[i++] = 0; // 跳到头都没出来
            }
        }
    }
};
```
I will describe its core ideas based on this code.
### Core Idea 1: Assigning Values to next Indices
The generation of the next array is the core part of the KMP algorithm. It records the length of the longest equal prefix and suffix for each position in the pattern string. With this array, the KMP algorithm can quickly skip unnecessary comparisons when a match fails, thereby improving efficiency. Therefore, the values stored at different indices of the next array are carefully designed:
* next[i] = x: Corresponds one-to-one with needle[i]; x is the length of the longest equal prefix and suffix of the substring before needle[i], and the prefix/suffix cannot be the entire preceding string.
* next[0] = -1: The first character has no prefix or suffix, so a negative value is stored to indicate this.
* next[1] = 0: The prefix and suffix cannot be the entire string, so it is 0.
* The same logic applies for subsequent indices.

### Core Idea 2: How the Main KMP Logic Uses next to Accelerate Matching
```c++
    while (x < n && y < m) {
        if (haystack[x] == needle[y]) {
            x++;
            y++;
        } else if (y == 0) {   
            x++;
        } else {
            y = next[y];
        }
    }
```
Suppose we have a substring `aabaaf`, and we generate the next array for `aabaaf`:
* next[0] = -1
* next[1] = 0
* next[2] = 1
* next[3] = 0
* next[4] = 1
* next[5] = 2
It is not difficult to see that: \
 When y = 4, the string before needle[4] is: `aaba`. needle[next[4] - 1] is the same as needle[3]. At this point, we can change the comparison from needle[4] with haystack[x] to needle[next[4]] (needle[1]) with haystack[x], thus avoiding the comparison with needle[0] and improving speed.


### Core Idea 3: Using cn to Accelerate next Array Generation
```c++
    while (i < m) {
        if (needle[i - 1] == needle[cn]) {  // 从needle中的下标为2 - 1开始, 开始为neeedle[1] == needle[0]相等的情况下
            // 后面为和上一个值相等的情况
            next[i++] = ++cn;                       // 长度最多比之前多一个, 在与之前最大位置的值相等的情况下
        } else if (cn > 0) {   // cn > 0 : 当前位置还能往前蹦
            cn = next[cn];
        } else {
            next[i++] = 0; // 跳到头都没出来
        }
    }
```
Why can cn accelerate the generation of the next array: \
next[0] = -1 and next[1] = 0 have already been initialized and are not included.
When needle[i - 1] equals needle[cn], the value stored in next[i] is one more than before, because when they are equal, the prefix and suffix can both be extended by one. When they are not equal, if cn > 0, there may exist a prefix that matches the current suffix, so we can replace the suffix with the prefix, achieving accelerated generation of next.


## FAQ
### Why not initialize the array with next[0] = 0, next[1] = 1?
next[0] = 0, next[1] = 1 is the standard version of KMP, while next[0] = -1, next[1] = 0 is the `failure function` version, which can better handle the index relationships of next and can use cn to accelerate the construction of the next array. It is superior to the standard version.
