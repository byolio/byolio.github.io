## What is the Fibonacci Sequence?
The `Fibonacci sequence` (also known as the golden ratio sequence) is a sequence introduced by the Italian mathematician Leonardo Fibonacci using the example of rabbit breeding. Its values are: `1, 1, 2, 3, 5, 8, 13, 21, 34...` Mathematically, this sequence is defined by the following recurrence:
`F(0)=0`, `F(1)=1`, `F(n)=F(n-1)+F(n-2) (n ≥ 2, n ∈ N*)`.

## Algorithm Implementations of the Fibonacci Sequence

### Recursive Implementation
This is the most straightforward implementation that comes to mind for most people. It is the simplest and easiest to write:
```c++
class Solution {
public:
    int fib(int n) {
        if (n == 0)
        {
            return 0;
        }
        if (n == 1)
        {
            return 1;
        }
        return fib(n - 1) + fib(n - 2);
    }
};
```

![fib](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/fib.png)
Because its time complexity is `O(2^n)`, it appears very slow in the image, so I strongly recommend against using it.

### Dynamic Programming Implementation
This is a space-for-time approach with a time complexity of `O(n)`. It has three implementation methods:

1. Bottom-up
2. Top-down
3. Space-optimized (top-down)


*1. Bottom-up*
```c++
class Solution {
public:
    vector<int> dp;  // Reverse dp processing
    int f(int i)
    {
        if (i == 0)
        {
            return 0;
        }
        if (i == 1)
        {
            return 1;
        }
        if (dp[i] != -1)
        {
            return dp[i];
        }
        int ans = f(i - 1) + f(i - 2);
        dp[i] = ans;
        return ans;
    }
    int fib(int n) {
        dp = vector<int>(n + 1, -1);
        return f(n);
    }
};
```

![dp1](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/dp1.png)


*2. Top-down*
```c++
class Solution {
public:
    vector<int> dp;
    int fib(int n) {
        if (n == 0)
        {
            return 0;
        }
        if (n == 1)
        {
            return 1;
        }
        dp = vector<int>(n + 1, 0);
        dp[1] = 1;
        for (int i = 2; i <= n; ++i)  // Use for loop for forward processing
        {
            dp[i] = dp[i - 1] + dp[i - 2];
        }
        return dp[n];
    }
};
```

![dp2](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/dp2.png)


*3. Space-optimized (top-down)*
```c++
class Solution {
public:
    int lastLast, last;    // Two variables replace dp to optimize space
    int fib(int n) {
        if (n == 0)
        {
            return 0;
        }
        if (n == 1)
        {
            return 1;
        }
        lastLast = 0;
        last = 1;
        for (int i = 2, tmp; i <= n; ++i)
        {
            tmp = last;
            last = last + lastLast;
            lastLast = tmp;
        }
        return last;
    }
};
```

![dp3](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/dp3.png)

These three methods: \
The first uses recursion + reverse dp \
The second uses forward dp \
The third uses two variables to replace dp for space optimization

Regardless of the method, the time complexity is `O(n)`, and in data testing, `0ms` poses no challenge. The implementation code is not difficult and is the most commonly used approach by participants in algorithm competitions.

### Matrix Fast Exponentiation Implementation
This is probably the most difficult part of this chapter. Its time complexity is `O(logn)`, making it one of the fastest algorithms for computing the Fibonacci sequence. The implementation is as follows:
```c++
class Solution {
public:
    // Matrix multiplication
    vector<vector<long long>> mult(vector<vector<long long>>& a, vector<vector<long long>>& b) {
        vector<vector<long long>> res(2, vector<long long>(2, 0));
        // Operation rules for two 2D matrices
        res[0][0] = a[0][0] * b[0][0] + a[0][1] * b[1][0];
        res[0][1] = a[0][0] * b[0][1] + a[0][1] * b[1][1];
        res[1][0] = a[1][0] * b[0][0] + a[1][1] * b[1][0];
        res[1][1] = a[1][0] * b[0][1] + a[1][1] * b[1][1];
        return res;
    }

    // Matrix fast exponentiation
    vector<vector<long long>> matrixPow(vector<vector<long long>> a, int n) {
        vector<vector<long long>> result = { {1, 0}, {0, 1} }; // Identity matrix
        while (n > 0) {
            if (n & 1) {
                result = mult(result, a);
            }
            a = mult(a, a);
            n >>= 1;
        }
        return result;
    }

    // Compute the nth Fibonacci number
    int fib(int n) {
        if (n == 0) return 0;
        vector<vector<long long>> base = { {1, 1}, {1, 0} };
        vector<vector<long long>> result = matrixPow(base, n - 1);
        return result[0][0];
    }
};
```

![matrix](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/matrix.png)
Note: Because its time complexity is `O(logn)`, and `O(n)` already achieves `0ms`, its superiority is not demonstrated in the test.

Since constructing the matrix incurs overhead, its superior time complexity only becomes apparent with large data. Therefore, for small data sizes, it may actually be slower than other implementations (for small data, the dynamic programming approach is better).
