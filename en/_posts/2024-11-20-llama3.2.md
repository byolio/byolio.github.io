> This article aims to build the local open-source large model Ollama 3.2.

Note: The entire process requires a VPN to download the Ollama 3.2 model and pull Docker images, and you need to have some technical foundation.

## What is Ollama 3.2
`Ollama 3.2` is an open-source multimodal large language model developed by `Meta`, capable of processing both text and image inputs. Its latest version `3.2` was released on `September 25, 2024`. The model series includes four versions: 1B, 3B, 11B, and 90B, with parameter sizes ranging from `1 billion` to `90 billion`. Among them, the `11B` and `90B` versions support image reasoning tasks, such as image understanding and visual reasoning; the `1B` and `3B` versions are pure text models, more suitable for local operation on edge devices and mobile devices. \
Deploying open-source models locally is free and can be used unlimited times, serving as an assistant for daily study and work.

## What Hardware Configuration Does Ollama 3.2 Require
This time, I will deploy the `3B` version of `Ollama 3.2`, which is the most suitable version for laptop hardware. Although `Meta` has not disclosed the hardware configuration required for local installation, based on the current model's parameter size and common practices, the hardware requirements are as follows: \
The `3B` version does not require a `GPU` and can also be installed as a `CPU` version. However, after testing, I found that the `CPU` version runs very slowly, so it is recommended to choose the `GPU` version for installation. For video memory, I recommend at least `8GB`; if the configuration is really insufficient, `4GB` can also work, but the same issue persists—the running speed will be very slow.

Information about other versions is as follows:
1. The `1B` version has lower configuration requirements than the `3B` version, but they are generally similar. However, its parameters are too few, leading to poor accuracy, so it is not recommended.
2. The `11B` version must be downloaded as the `GPU` version. I recommend at least `22GB` of video memory and a minimum of `16GB` of RAM.
3. The `90B` version must be downloaded as the `GPU` version. I recommend at least `40GB` of video memory and a minimum of `64GB` of RAM.

## How to Install Ollama 3.2
After understanding the hardware requirements for `Ollama 3.2 3B` and confirming the need for local AI deployment, we can start preparing to install `Ollama 3.2 3B`. \
To make it more convenient for daily use, I decided to deploy the version with a `web UI` on `Docker`, so I can access the `web UI` through a browser to use `Ollama 3.2`, instead of deploying it directly on the local computer and using it via `cmd command line`. \
I will use the `wsl + docker` method to achieve this.

### Download WSL
I decided to deploy Docker in the Linux subsystem of Win11, so I first installed `WSL`. (It is recommended to upgrade Win11 to the Professional version to install WSL. I am using a Win11 computer upgraded to the Professional version to demonstrate the installation process.) \
The process is as follows: \
Open `Control Panel` -> `Programs` -> `Turn Windows features on or off` -> Check `Windows Subsystem for Linux` -> Click `OK` -> Click `Restart`. \
After restarting the computer, `WSL` is installed. \
If you are unsure whether it is installed, you can open cmd and enter the command `wsl --version` to check if the installation was successful.

### Install Docker
I use the Windows version of `Docker Desktop`. [Click here to download directly](https://www.docker.com/products/docker-desktop/). Select `Download for Windows-AMD64` to download `Docker Desktop`, then install it. Just make sure to check `Use WSL 2 instead of Hyper-V (recommended)` to use WSL. \
Next, start Docker Desktop. You do not need to log in; you can proceed with the following steps for deployment.

### Deploy Open Web UI
Before deploying `Ollama 3.2`, I first download `Open Web UI`. Here, I download the `GPU` version of `Open Web UI`. \
`Open Web UI` is an open-source project that provides a user-friendly interface, allowing developers and data scientists to use machine learning and deep learning models in a browser without writing code or complex setups. The deployment process is as follows: \
Open cmd and enter the command:
```bash
docker run -d -p 3000:8080 --gpus=all -v ollama:/root/.ollama -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:ollama
```
Here is a brief explanation: \
1. docker run: Creates an instance; if there is no local image, it pulls from `Docker Hub`. \
2. -d: Runs the instance in the background. \
3. -p: Maps the container port to the host. \
4. --gpus=all: Allows the container to access all GPUs. \
5. -v: Volume mount or directory mount; here it is a volume mount. \
6. --name: Container name. \
7. --restart always: Container restart policy; set to `always` means the container always restarts after an unexpected stop. \
8. ghcr.io/open-webui/open-webui:ollama: Image name; here it is the `ollama` version of `Open Web UI`. \
(You must specify the version; otherwise, it will pull the latest version.)

### Start Open Web UI
Open a browser and enter `localhost:3000` in the address bar, then register an account and password to access `Open Web UI`.

### Download Ollama 3.2 3B
Click the user avatar in the bottom left corner -> Click `Admin Panel` \
![user](https://cdn.jsdelivr.net/gh/byolio/newtc@main/img/ollama-user.png)
Then click `Settings` -> `Models`, and in the following box:
![alt text](https://cdn.jsdelivr.net/gh/byolio/newtc@main/img/ollama-img.png)
Enter:
```text
llama:3b
```
Then click the download button next to it to pull the model. A VPN is required throughout the process. Wait for the download progress bar to finish, then restart `Open Web UI`, select the model, and it will be ready to use.
