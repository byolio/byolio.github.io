## What is Subarray Sum
`Subarray Sum`, its core idea is to decompose the original problem into a series of overlapping subproblems, solve these subproblems step by step and store their results to avoid repeated calculations and improve algorithm efficiency. Essentially, it is an algorithm idea of `trading space for time`. In solving problems, it also uses the idea of `Dynamic Programming` (DP), and often uses `Prefix Sum` and `Suffix Sum` techniques for efficient computation.

## Implementing Ordinary Subarray Sum
It is mainly implemented using DP ideas, with two main approaches:
(Test link: [https://leetcode.cn/problems/maximum-subarray/](https://leetcode.cn/problems/maximum-subarray/)):
1. Ordinary DP implementation
```c++
class Solution {
public:
    vector<int> dp;
    int maxSubArray(vector<int>& nums) {
        int n = nums.size();
        dp = vector<int>(n, 0);     //1 <= nums.length <= 105
        dp[0] = nums[0];
        int ans = dp[0];
        for (int i = 1; i < n; ++i)
        {
            dp[i] = max(nums[i], dp[i - 1] + nums[i]);
            ans = max(ans, dp[i]);
        }
        return ans;
    }
};
```
Analysis:
* Since nums.length >= 1, we can omit the corresponding boundary checks.
* Initialize dp[0] = nums[0] so that starting from i = 1, dp[i - 1] will not go out of bounds.
* During the DP process, record the maximum value simultaneously to avoid traversing the dp array again at the end.
* ans is initialized to dp[0] to handle the case where nums has only one element or the maximum includes nums[0].

Comment: The time complexity has been optimized to O(n) using DP, but the space complexity is O(n), which can be further optimized.

1. Space-optimized DP implementation
```c++
// Space compression
class Solution {
public:
    int pre;   // 1 <= nums.length <= 105
    int maxSubArray(vector<int>& nums) {
        int n = nums.size();
        pre = nums[0];
        int ans = nums[0];
        for (int i = 1; i < n; ++i)
        {
            pre = max(nums[i], pre + nums[i]);
            ans = max(ans, pre);
        }
        return ans;
    }
};
```
Analysis:
* Use pre instead of the dp array to record dp[i - 1], optimizing space complexity to O(1).

Comment: The space complexity has been optimized to O(1), and the time complexity is also O(n). This implementation approach is optimal.

## Implementing Non-Circular Subarray Sum
It is also mainly implemented using DP ideas:
(Test link: [https://leetcode.cn/problems/house-robber/](https://leetcode.cn/problems/house-robber/)):
```c++
#include<algorithm>
class Solution {
public:
    vector<int> dp;
    int rob(vector<int>& nums) {
        int n = nums.size();
        // Handle special cases
        if (n == 1)
        {
            return nums[0];
        }
        if (n == 2)
        {
            return max(nums[0], nums[1]);
        }
        // DP initialization
        dp = vector<int>(n, 0);
        dp[0] = nums[0];
        dp[1] = max(nums[0], nums[1]);

        // Main logic
        for (int i = 2; i < n; ++i)
        {
            dp[i] = max(dp[i - 1], max(dp[i], dp[i - 2] + nums[i]));
        }
        return dp[n - 1];
    }
};
```
Analysis:
* Handle `dp[1] = max(nums[0], nums[1])` separately for the cases of robbing house 1 and house 2.
* Use `dp[i] = max(dp[i - 1], max(dp[i], dp[i - 2] + nums[i]))` to determine the maximum between robbing house i and not robbing house i, taking the larger value as the current maximum.

## Implementing Circular Subarray Sum
Test link: [https://leetcode.cn/problems/maximum-sum-circular-subarray/](https://leetcode.cn/problems/maximum-sum-circular-subarray/)
This problem has two cases:
1. The maximum subarray is in the middle; directly use the ordinary subarray sum to find the maximum.
2. The maximum subarray requires including both the beginning and end of the array; then use the total sum minus the minimum subarray sum in the middle to get the maximum.
```c++
#include<climits>
#include<algorithm>
class Solution {
public:
    int maxSubarraySumCircular(vector<int>& nums) {
        int n = nums.size();
        int all = nums[0];
        int maxSum = nums[0];
        int minSum = nums[0];
        for (int i = 1, maxPre = nums[0], minPre = nums[0]; i < n; ++i)
        {
            all += nums[i];
            maxPre = max(nums[i], maxPre + nums[i]);   // when choosing the maximum
            maxSum = max(maxSum, maxPre);

            minPre = min(nums[i], minPre + nums[i]);   // when choosing the minimum
            minSum = min(minSum, minPre);   
        }
        return all == minSum ? maxSum : max(maxSum, all - minSum);  
        // all == minSum: all numbers are negative, but cannot return an empty array, so return maxSum
    }
};
```
Analysis:
* `all - minSum` is the case where the array needs to include both ends; `maxSum` is the case where it does not.
* `all == minSum` is the case where all numbers are negative; but since the return value cannot be an empty array, we can only return `maxSum`.

## Implementing Subarray Sum Using Prefix and Suffix Sum
I think this part is relatively difficult to understand. It is mainly used for problems that require the maximum sum of segmented subarrays.
Test link: [https://leetcode.cn/problems/maximum-sum-of-3-non-overlapping-subarrays/](https://leetcode.cn/problems/maximum-sum-of-3-non-overlapping-subarrays/)
```c++
class Solution {
public:
    vector<int> sums;                       // cumulative sum starting from position i for k consecutive elements
    vector<int> prefix;
    vector<int> suffix;
    vector<int> maxSumOfThreeSubarrays(vector<int>& nums, int k) {
        int n = nums.size();
        sums = vector<int>(n);
        for (int l = 0, r = 0, sum = 0; r < n; ++r)
        {
            sum += nums[r];
            if (r - l + 1 == k)
            {
                sums[l] = sum;
                sum -= nums[l++];
            }
        }
        // prefix : index of the maximum subarray sum in the range 0 ~ i
        prefix = vector<int>(n, 0);
        prefix[0] = 0;
        for (int l = 1, r = k; r < n; ++l, ++r)
        {
            if (sums[l] > sums[prefix[r - 1]])  // if the current position has a larger subarray sum than the previous best
            {
                prefix[r] = l;                  
            }else{
                prefix[r] = prefix[r - 1];      
            }
        }
        // suffix : index of the maximum subarray sum in the range i ~ n - 1, the index is the starting index of the suffix
        suffix = vector<int>(n);
        suffix[n - k] = n - k;
        for (int l = n - 1 - k; l >= 0; --l)
        {
            if (sums[l] >= sums[suffix[l + 1]])  // note: >= is used because the problem requires the lexicographically smallest result if multiple results exist.
            {
                suffix[l] = l;
            }else{
                suffix[l] = suffix[l + 1];
            }
        }
        int a, b, c, maxVal = 0;
        for (int p, s, i = k, j = 2 * k - 1, sum; j < n - k; ++i, ++j)
        {
            p = prefix[i - 1];
            s = suffix[j + 1];
            sum = sums[p] + sums[i] + sums[s];
            if (sum > maxVal)
            {
                a = p;
                b = i;
                c = s;
                maxVal = sum;
            }
        }
        return {a, b, c};
    }
};
```
Analysis:
* Uses the DP idea of prefix and suffix sums to record starting indices. When at position i, we need to find the index of the maximum subarray sum in `0 ~ i - 1` and the index of the maximum subarray sum in `i + k ~ n - 1`. Then use `sum = sums[p] + sums[i] + sums[s];` to find the maximum sum of three subarrays.
* Note the use of `>=` because if there are multiple results, the lexicographically smallest one should be returned.

Comment: The time complexity is O(n), which is a very good implementation.

## Summary
1. The core idea of subarray sum is to decompose the original problem into a series of overlapping subproblems, solve these subproblems step by step and store their results to avoid repeated calculations and improve algorithm efficiency. Essentially, it is an algorithm idea of `trading space for time`.
2. In solving problems, it also uses the idea of `Dynamic Programming` (DP), and often uses `Prefix Sum` and `Suffix Sum` techniques for efficient computation.
