## What is Subarray Sum
`Subarray Sum` is a concept whose core idea is to decompose the original problem into a series of overlapping subproblems, solve these subproblems step by step, and store their results to avoid redundant calculations, thereby improving algorithm efficiency. It is essentially an algorithmic approach of `trading space for time`. When solving problems, it also utilizes the idea of `Dynamic Programming` (DP) and often employs `Prefix Sum` and `Suffix Sum` techniques for efficient computation.

## Implementing Basic Subarray Sum
It is mainly implemented using the DP approach, with two primary methods:
(Test link: [https://leetcode.cn/problems/maximum-subarray/](https://leetcode.cn/problems/maximum-subarray/)):
1. Basic DP Implementation
```c++
class Solution {
public:
    vector<int> dp;
    int maxSubArray(vector<int>& nums) {
        int n = nums.size();
        dp = vector<int>(n, 0);     //1 <= nums.length <= 105
        dp[0] = nums[0];
        int ans = dp[0];
        for (int i = 1; i < n; ++i)
        {
            dp[i] = max(nums[i], dp[i - 1] + nums[i]);
            ans = max(ans, dp[i]);
        }
        return ans;
    }
};
```
Analysis: 
* Since `nums.length >= 1`, the corresponding boundary checks can be omitted.
* `dp[0] = nums[0]` initializes the DP array, allowing the loop to start from `i = 1` without `dp[i - 1]` going out of bounds.
* During the DP process, the maximum value is recorded simultaneously to avoid traversing the `dp` array again at the end.
* `ans` is initialized to `dp[0]` to handle cases where `nums` has only one element or the maximum value includes `nums[0]`.

Comment: The time complexity has been optimized to O(n) using DP, but the space complexity is O(n), which can be further optimized.

1. Space-Optimized DP Implementation
```c++
// Space compression
class Solution {
public:
    int pre;   // 1 <= nums.length <= 105
    int maxSubArray(vector<int>& nums) {
        int n = nums.size();
        pre = nums[0];
        int ans = nums[0];
        for (int i = 1; i < n; ++i)
        {
            pre = max(nums[i], pre + nums[i]);
            ans = max(ans, pre);
        }
        return ans;
    }
};
```
Analysis:
* Using `pre` instead of the `dp` array to record `dp[i - 1]` optimizes the space complexity to O(1).

Comment: The space complexity has been optimized to O(1), and the time complexity is also O(n). This implementation approach is optimal.

## Implementing Non-Circular Subarray Sum
This is also mainly implemented using DP:
(Test link: [https://leetcode.cn/problems/house-robber/](https://leetcode.cn/problems/house-robber/)):
```c++
#include<algorithm>
class Solution {
public:
    vector<int> dp;
    int rob(vector<int>& nums) {
        int n = nums.size();
        // Handle special cases separately
        if (n == 1)
        {
            return nums[0];
        }
        if (n == 2)
        {
            return max(nums[0], nums[1]);
        }
        // DP initialization
        dp = vector<int>(n, 0);
        dp[0] = nums[0];
        dp[1] = max(nums[0], nums[1]);

        // Main logic
        for (int i = 2; i < n; ++i)
        {
            dp[i] = max(dp[i - 1], max(dp[i], dp[i - 2] + nums[i]));
        }
        return dp[n - 1];
    }
};
```
Analysis:* The separate handling `dp[1] = max(nums[0], nums[1])` deals with the cases of robbing house 1 or house 2.
* Using `dp[i] = max(dp[i - 1], max(dp[i], dp[i - 2] + nums[i]))` determines the maximum between robbing house `i` and not robbing house `i`, taking the larger value as the current maximum.

## Implementing Circular Subarray Sum
Test link: [https://leetcode.cn/problems/maximum-sum-circular-subarray/](https://leetcode.cn/problems/maximum-sum-circular-subarray/)
This problem has two scenarios: 
1. The maximum subarray is in the middle; simply use the basic subarray sum to find the maximum.
2. The maximum subarray requires including both the beginning and end of the array; then the maximum is the total sum minus the minimum subarray sum in the middle.
```c++
#include<climits>
#include<algorithm>
class Solution {
public:
    int maxSubarraySumCircular(vector<int>& nums) {
        int n = nums.size();
        int all = nums[0];
        int maxSum = nums[0];
        int minSum = nums[0];
        for (int i = 1, maxPre = nums[0], minPre = nums[0]; i < n; ++i)
        {
            all += nums[i];
            maxPre = max(nums[i], maxPre + nums[i]);   // When choosing the maximum
            maxSum = max(maxSum, maxPre);

            minPre = min(nums[i], minPre + nums[i]);   // When choosing the minimum
            minSum = min(minSum, minPre);   
        }
        return all == minSum ? maxSum : max(maxSum, all - minSum);  
        // all == minSum : All elements are negative, but the result cannot be an empty array, so return maxSum
    }
};
```
Analysis:
* `all - minSum` represents the case where the subarray includes both ends, while `maxSum` represents the case where it does not.
* `all == minSum` indicates all elements are negative; since the result cannot be an empty array, only `maxSum` can be returned.

## Implementing Subarray Sum Using Prefix and Suffix Sums
This part is relatively difficult to understand and is mainly used for problems involving maximizing the sum of multiple non-overlapping subarrays.
Test link: [https://leetcode.cn/problems/maximum-sum-of-3-non-overlapping-subarrays/](https://leetcode.cn/problems/maximum-sum-of-3-non-overlapping-subarrays/)
```c++
class Solution {
public:
    vector<int> sums;                       //  Cumulative sum of k elements starting from position i
    vector<int> prefix;
    vector<int> suffix;
    vector<int> maxSumOfThreeSubarrays(vector<int>& nums, int k) {
        int n = nums.size();
        sums = vector<int>(n);
        for (int l = 0, r = 0, sum = 0; r < n; ++r)
        {
            sum += nums[r];
            if (r - l + 1 == k)
            {
                sums[l] = sum;
                sum -= nums[l++];
            }
        }
        // prefix : Index of the maximum subarray sum from 0 to i
        prefix = vector<int>(n, 0);
        prefix[0] = 0;
        for (int l = 1, r = k; r < n; ++l, ++r)
        {
            if (sums[l] > sums[prefix[r - 1]])  // If the current position's subarray sum is greater than the previous maximum
            {
                prefix[r] = l;                  
            }else{
                prefix[r] = prefix[r - 1];      
            }
        }
        // suffix : Index of the maximum subarray sum from i to n-1, the index is the starting index of the suffix
        suffix = vector<int>(n);
        suffix[n - k] = n - k;
        for (int l = n - 1 - k; l >= 0; --l)
        {
            if (sums[l] >= sums[suffix[l + 1]])  // Note the >= here; the problem requires returning the lexicographically smallest result if multiple exist.
            {
                suffix[l] = l;
            }else{
                suffix[l] = suffix[l + 1];
            }
        }
        int a, b, c, maxVal = 0;
        for (int p, s, i = k, j = 2 * k - 1, sum; j < n - k; ++i, ++j)
        {
            p = prefix[i - 1];
            s = suffix[j + 1];
            sum = sums[p] + sums[i] + sums[s];
            if (sum > maxVal)
            {
                a = p;
                b = i;
                c = s;
                maxVal = sum;
            }
        }
        return {a, b, c};
    }
};
```
Analysis:
* This uses the DP idea of prefix and suffix sums to record starting indices. When at position `i`, it needs to find the index of the maximum subarray sum from `0 to i-1` and from `i+k to n-1`. Then, using `sum = sums[p] + sums[i] + sums[s]`, it finds the maximum sum of three subarrays.
* Note the use of `>=` in the comparison, as the problem requires returning the lexicographically smallest result if multiple exist.

Comment: The time complexity is O(n), making it a very good implementation approach.
