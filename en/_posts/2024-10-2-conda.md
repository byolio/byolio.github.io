## What is Conda?

Simply put: `Conda` is a general-purpose package manager and environment manager, originally developed by `Anaconda`. It can manage not only `Python` packages but also supports other programming languages (such as `R`, `Julia`, etc.). Therefore, Conda's advantage lies in its ability to manage libraries, tools, and dependencies across languages, freeing developers from being limited to a single programming language.

Unlike `pip`, which focuses on managing `Python` packages, `Conda` aims to manage the entire development environment, including programming languages, operating system libraries, and tools. This means it not only manages `Python` packages and installs corresponding `Python` dependencies but can also install system-level dependencies and libraries for other languages (such as `C++` compilers or `C` language libraries).

## Why Install Conda?

Some people mention the important role of `Conda` in data analysis and machine learning, which is indeed one of the reasons for its widespread adoption. However, besides this factor, another key advantage of `Conda` lies in its `dependency management` and handling of `built-in third-party dependency files`:

1. Excellent dependency management: `Conda`'s package management mechanism excels at handling complex dependency relationships. When you install a package, it automatically resolves and installs all its dependencies, ensuring version compatibility. This greatly reduces the risk of "dependency hell" (i.e., conflicts between package dependencies), which is especially important when managing multiple packages and using them in different environments.
2. Built-in third-party dependency files: Unlike `pip`, `Conda` provides precompiled binary packages. This means many packages that depend on specific system libraries (e.g., packages requiring a `C` compiler) do not need an additional compilation process; `Conda` automatically installs these dependency libraries. For some common third-party dependency libraries (such as `numpy`, `scipy`, `pandas`, etc.), `Conda` provides optimized compiled versions that can be installed directly, greatly reducing the hassle of manual compilation and environment configuration for developers.\
(Because `Conda` includes a large number of `Python` third-party dependency libraries, it can meet the needs of most users. Unlike `pip`, which requires using a proxy or other methods to install these dependencies, it solves the problem for those who cannot use such methods. This is also why I recommend it.)

## How to Install Conda

After all that talk, it's time for the main event. Let's see how to install `Conda`.

1. Download the `Anaconda` installer
   First, we need to download the `Conda` installer. You can download it from the [`Anaconda` official website](https://www.anaconda.com/download/success) (this link points directly to the download area).
   ![condaDownload](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/condaDownload.png)
   Choose the appropriate installer for your operating system. Since I use `Windows`, I select the one on the far left.
2. Install `Anaconda`
   After downloading the installer, we need to install `Anaconda`. The installation process is very simple; just follow the prompts.
   ![anacondaInstall](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/anacondaInstall.png)
3. Configure `Conda` environment variables in `PyCharm`
   First, go to the `Settings` interface in `PyCharm`, click on `Project` and then `Python Interpreter`. (There are many ways to access it; here we only cover one.)
   ![pycharmSetting](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/pycharmSetting.png)
   Click `Add Interpreter`, select `Conda Environment`, then click the folder icon button.
   ![pycharmConda](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/CondaEnv.jpg)
   Find the path where you installed `Anaconda`, import `conda.exe`, and finally click the `Load Environments` button.
   For example, my path is here:
   ![pycharmCondaPath](https://cdn.jsdelivr.net/gh/byolio/mytc@main/img/condaPath.png) \
   Click `Create new environment`, enter the environment name and its Python version, then click `OK` to create a new `Conda` environment.

## Summary

`Conda` is an excellent cross-platform package manager and environment manager that helps developers easily manage the entire development environment, including `Python` packages, operating system libraries, and tools. If you haven't installed `Conda` yet, now is the best time to do so.
